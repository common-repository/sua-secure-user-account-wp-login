SUA – Secure User Account WordPress Login is a plugin that will allow you to secure your account 

== Description ==

<p>
SUA – Secure User Account WordPress Login is a plugin that will allow you to secure your account and change your personal information in a quick and easy way <a href="https://www.ixlaf.com/demos/secure-user-account-wordpress-login/">Online Demo</a>
</p>
<ul>
        <li><a href="http://www.ixlaf.com/docs/sua/installation/">Installation</a> : The plugin is easy to install like any other WordPress plugins</li>
        <li><a href="http://www.ixlaf.com/docs/sua/configuration/">Configuration</a> : After the installation of the plugin, you just have to create a new page with the shortcode [sua] and add an administrator with a secure username (case insensitive) and password(at least 1 uppercase character (A-Z)
at least 1 lowercase character (a-z)
at least 1 digit (0-9)
at least 1 special character
at least 10 characters
no more than 2 identical characters)</li>
        <li><a href="http://www.ixlaf.com/docs/sua/sign-in/">Sign In</a> : The authentication is secured by using the new Google captcha (reCAPTCHA) which is safer than blocking the IP addresses to prevent brute-force attacks</li>
        <li><a href="http://www.ixlaf.com/docs/sua/sign-up/">Sign Up</a> : New user needs a username, email address and a password to register, the plugin reserve usernames and email addresses only after the validation of the account</li>
        <li><a href="http://www.ixlaf.com/docs/sua/account-history/">Account History</a> : History of all connections made on your account with all the necessary information to identify the user (browser, country, city, ip address)</li>
        <li><a href="http://www.ixlaf.com/docs/sua/account-home/">Account Home</a> : The user space is well presented, and the navigation is done in a modern manner, without reloading the page by using a secured single-page application (error messages, input validation, logging, clickjacking defense, cross-site request forgery (csrf) prevention, cross-site scripting (xss) prevention)</li>
        <li><a href="http://www.ixlaf.com/docs/sua/change-password/">Change Password</a> : The password is easy to edit, and you can change the number of attempts that a user can do before the automatic logout</li>
        <li><a href="http://www.ixlaf.com/docs/sua/delete-account/">Delete Account</a> : All users can delete their accounts and you can change the number of days before the deletion</li>
        <li><a href="http://www.ixlaf.com/docs/sua/email-notifications/">Email Notifications</a> : You can enable email notifications by using the primary email address, recovery address or both, for all critical operations made on the user account</li>
        <li><a href="http://www.ixlaf.com/docs/sua/ip-verification/">IP Verification</a> <strong>*Only in Premium Version</strong> : To enable the IP verification you just have to activate this option on your account and you will receive a mail with a code every time you will login to your account from a different location</li>
        <li><a href="http://www.ixlaf.com/docs/sua/mails/">Mails</a> : All mail sent by the plugin are editable</li>
        <li><a href="http://www.ixlaf.com/docs/sua/multifactor-authentication/">Multifactor Authentication</a> <strong>*Only in Premium Version</strong> : The multi-factor authentication (MFA)  has become necessary for all secure connections, that’s why the plugin offers this option by using a grid that you can print or save to confirm your connections</li>
        <li><a href="http://www.ixlaf.com/docs/sua/notifications/">Notifications</a> : All notifications sent by the plugin are editable</li>
        <li><a href="http://www.ixlaf.com/docs/sua/personal-information/">Personal Information</a> : The user can change his personal information easily, and each time that the primary email address is changed the user must validate it before it will be accepted</li>
        <li><a href="http://www.ixlaf.com/docs/sua/pin-code/">Pin Code</a> : The pin code is a 4 digit code to enhance the security of the account, to avoid using the password for each change and to require re-authentication for sensitive features</li>
        <li><a href="http://www.ixlaf.com/docs/sua/recovery/">Recovery</a> : The plugin implements a very secured password recovery mechanism to prevent spam and hacking attempts</li>
        <li><a href="http://www.ixlaf.com/docs/sua/recovery-email/">Recovery Email</a> : Each user can have a recovery address which can be used in the recovery of his account if he can’t use his primary address</li>
        <li><a href="http://www.ixlaf.com/docs/sua/security-questions/">Security Questions</a> : Security questions are used before the recovery of an account to confirm the change of the password, answers are encrypted and the user can either create their own questions or use canned questions that are provided by the plugin</li>
        <li><a href="http://www.ixlaf.com/docs/sua/token-verification/">Token Verification</a> : The tokens used by the plugin are unique and hard to guess and they have a lifetime that you can easily change</li>
</ul>
